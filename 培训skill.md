# 培训需求skill开发指南

**调整：**提取的需求文档等md及json文件，命名应该同文件里的编号一致，不应该再用原始的Excel名字了，免得后面对应不上


总共6个skill

1 培训流程总控器
2 需求处理器
3 需求汇总器
4 课程解析器
5 课程能力构建器
6 需求课程匹配器

其中第一个不建议使用，中间容错率太低

每个 Skill 我都会按统一结构描述：

Skill 目标
输入
输出
使用的 Prompt
执行步骤
处理规则
失败处理
返回结果

这样 Agent 能够比较容易执行。

# Skill 1：培训流程总控器

## Skill 名称

training_pipeline_orchestrator

## Skill 目标

负责串联整个培训分析流程。
它不处理具体业务数据，只负责按顺序调用其他 Skill，并汇总最终结果。

## 输入

无固定输入参数。
默认从系统目录读取数据。

需求来源目录：

```
01_培训需求库/01_原始Excel/
```

课程来源目录：

```
02_课程大纲库/01_原始课纲/
```

## 输出

最终输出文件：

```
04_知识检索库/需求课程匹配结果/培训建议汇总报告.md
```

同时会生成中间数据：

需求 JSON
课程能力库
匹配结果 JSON

## 调用 Skill 顺序

执行顺序必须严格按下面顺序：

1 调用 `需求处理器`
2 调用 `需求汇总器`
3 调用 `课程解析器`
4 调用 `课程能力构建器`
5 调用 `需求课程匹配器`

## 执行步骤

步骤1
检查是否存在需求 Excel 文件。

路径：

```
01_培训需求库/01_原始Excel/
```

如果为空，停止流程。

步骤2
调用 Skill：

```
需求处理器
```

生成需求提取 JSON。

步骤3
调用 Skill：

```
需求汇总器
```

生成需求汇总池。

步骤4
检查课程课纲目录。

路径：

```
02_课程大纲库/01_原始课纲/
```

步骤5
调用 Skill：

```
课程解析器
```

生成课纲解析结果。

步骤6
调用 Skill：

```
课程能力构建器
```

生成课程能力库。

步骤7
调用 Skill：

```
需求课程匹配器
```

生成匹配结果。

步骤8
汇总匹配结果，生成培训建议报告。

## 返回结果

返回结构：

```json
{
  "pipeline_status": "completed",
  "processed_demands": 12,
  "processed_courses": 8,
  "generated_matches": 12,
  "report_path": "04_知识检索库/需求课程匹配结果/培训建议汇总报告.md"
}
```

# Skill 2：需求处理器

## Skill 名称

demand_processor

## Skill 目标

将培训需求 Excel 文件转换为结构化需求 JSON。

## 输入

Excel 文件目录：

```
01_培训需求库/01_原始Excel/
```

Excel 文件包含字段：

需求标题
需求来源部门
涉及岗位
问题描述
培训目标
业务影响

## 输出

Markdown 文件：

```
01_培训需求库/02_MD转换结果/
```

JSON 文件：

```
01_培训需求库/03_需求提取结果/
```

## 使用 Prompt

```
需求提取Prompt.md
```

## 执行步骤

步骤1
扫描目录：

```
01_培训需求库/01_原始Excel/
```

步骤2
调用脚本：

```
excel_to_md.py
```

将 Excel 转换为 Markdown。

输出：

```
02_MD转换结果/*.md
```

步骤3
读取 Markdown 内容。

步骤4
调用 Prompt：

```
需求提取Prompt
```

提取结构化需求信息。

步骤5
生成 JSON。

## 输出 JSON 示例

```json
{
  "需求ID": "DEMAND-2026-001",
  "需求标题": "门店销售主动销售能力不足",
  "需求来源部门": "销售部",
  "涉及岗位": ["门店销售"],
  "问题现象": "员工不会主动介绍产品",
  "能力类型": ["销售能力","客户沟通"],
  "培训目标": "提升主动销售能力",
  "需求紧急程度": "中",
  "需求关键词": ["主动销售","成交引导"],
  "是否适合培训解决": "是"
}
```

## 返回结果

```json
{
  "processed_files": 5,
  "generated_demands": 12,
  "output_directory": "01_培训需求库/03_需求提取结果/"
}
```

# Skill 3：需求汇总器

## Skill 名称

demand_aggregator

## Skill 目标

将多个需求 JSON 合并成需求池，并筛选重点需求。

## 输入

```
01_培训需求库/03_需求提取结果/*.json
```

## 输出

```
01_培训需求库/04_需求汇总池/
```

生成文件：

全部需求汇总.json
高优先级需求.json
待确认需求.json

## 执行步骤

步骤1
读取所有需求 JSON。

步骤2
合并需求列表。

步骤3
根据规则筛选高优先级需求。

筛选规则：

需求紧急程度 = 高
或
业务影响包含关键指标

步骤4
识别待确认需求。

筛选规则：

```
是否适合培训解决 = 部分适合
或
是否适合培训解决 = 待确认
```

步骤5
生成汇总 JSON。

## 输出 JSON 示例

```json
{
  "需求总数": 12,
  "高优先级需求数": 4,
  "待确认需求数": 2
}
```

# Skill 4：课程解析器

## Skill 名称

course_parser

## Skill 目标

解析讲师课纲文件，提取课程结构信息。

## 输入

课程文件目录：

```
02_课程大纲库/01_原始课纲/
```

支持文件类型：

pdf
pptx
docx
md

## 输出

```
02_课程大纲库/02_课纲解析结果/
```

## 使用 Prompt

```
课程解析Prompt.md
```

## 执行步骤

步骤1
扫描课程文件目录。

步骤2
读取课纲文本。

步骤3
调用 Prompt：

```
课程解析Prompt
```

步骤4
生成解析结果 Markdown。

## 输出示例

```
销售谈判技巧_解析结果.md
```

内容包括：

课程主题
适用对象
能力点
模块结构
教学方式

# Skill 5：课程能力构建器

## Skill 名称

course_builder

## Skill 目标

将课程解析结果转成标准课程能力对象。

## 输入

```
02_课程大纲库/02_课纲解析结果/
```

## 输出

```
02_课程大纲库/03_课程能力汇总库/
```

## 生成文件

课程结构卡片.md
课程摘要.md
能力标签.json
讲师版本映射.json

## 执行步骤

步骤1
读取课纲解析结果。

步骤2
生成课程结构卡片。

步骤3
生成课程摘要。

步骤4
生成能力标签 JSON。

步骤5
生成讲师映射。

步骤6
更新课程索引。

## 输出 JSON 示例

```json
{
  "课程ID": "COURSE-001",
  "课程名称": "销售谈判技巧",
  "能力类型": ["销售能力","客户沟通"],
  "核心能力点": ["需求挖掘","成交引导"],
  "适用岗位": ["门店销售"]
}
```

# Skill 6：需求课程匹配器

## Skill 名称

demand_course_matcher

## Skill 目标

根据培训需求，从课程能力库中匹配最合适课程。

## 输入

需求：

```
01_培训需求库/04_需求汇总池/高优先级需求.json
```

课程：

```
02_课程大纲库/03_课程能力汇总库/
```

## 使用 Prompt

```
需求课程匹配Prompt.md
```

## 输出

```
04_知识检索库/需求课程匹配结果/
```

文件：

匹配结果.json
匹配报告.md

## 执行步骤

步骤1
读取需求。

步骤2
加载课程能力库。

步骤3
根据能力类型、岗位、关键词筛选候选课程。

步骤4
调用 Prompt：

```
需求课程匹配Prompt
```

步骤5
生成匹配结果。

## 输出 JSON 示例

```json
{
  "需求ID": "DEMAND-2026-001",
  "推荐课程": [
    {
      "课程ID": "COURSE-001",
      "课程名称": "销售谈判技巧",
      "匹配分数": 0.91,
      "推荐讲师": ["张三"]
    }
  ]
}
```

# 六个 Skill 的整体关系

Agent 执行逻辑：

```
培训流程总控器
    ↓
需求处理器
    ↓
需求汇总器
    ↓
课程解析器
    ↓
课程能力构建器
    ↓
需求课程匹配器
```

每个 Skill 只依赖上一个 Skill 的输出文件。

---

# 最关键的设计原则

所有 Skill 不共享内存状态
只通过文件交换数据

例如：

需求处理器 输出 JSON
需求汇总器 读取 JSON
课程解析器 输出 Markdown
课程构建器 读取 Markdown

这样系统最稳定。
